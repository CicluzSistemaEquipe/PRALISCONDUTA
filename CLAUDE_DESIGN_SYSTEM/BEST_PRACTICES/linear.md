# Linear

> ⭐ **REGRA DE OURO:** extrair PRINCÍPIOS (tabelas, snappy, atalhos) —
> nunca copiar layout, identidade, paleta ou componentes. Se levar a algo que
> "não parece Pralís", descartar.

## Em uma frase
Linear é o padrão-ouro de **ferramenta produtiva**: interface tão rápida e responsiva
que parece uma extensão do pensamento, com atalhos para tudo e zero latência.

## Princípios a aprender
- **Snappy acima de tudo.** Respostas instantâneas (<100ms), optimistic UI, sem
  spinners onde se pode evitar. Velocidade percebida = qualidade percebida.
- **Teclado-primeiro.** Atalhos para navegar, criar, filtrar; `Cmd+K` para tudo.
  Power users voam sem tocar no mouse.
- **Tabelas/listas densas mas legíveis.** Linha = unidade de ação; hover revela ações.
- **Status como linguagem visual.** Estados (todo/doing/done) com cor+ícone consistentes.
- **Transições curtas e funcionais** (150–200ms), nunca decorativas.

## Como aplicar na Pralís
- ☀️ **Admin/CMS:** modelo direto para as **tabelas de colaboradores/relatórios** —
  hover com ações, filtros rápidos, densidade controlada, navegação por teclado.
- ☀️ **Dashboard:** status com cor+ícone (ativo/pendente/concluído) consistente em
  todo o admin; vire vocabulário visual reutilizável.
- ☀️ **CMS:** criação rápida de itens com atalhos; salvar otimista, feedback imediato.
- 🌙 **App:** herda a sensação *snappy* — transições curtas, nada de telas que travam.

## NÃO copiar
- Estética **dark cyber-frio roxo/cinza** — anti-calor Pralís. Admin é **claro**.
- Frieza de "ferramenta de engenheiro"; o admin Pralís é produtivo **e** acolhedor.
- Densidade extrema no app do colaborador (lá a régua é leveza, não throughput).
