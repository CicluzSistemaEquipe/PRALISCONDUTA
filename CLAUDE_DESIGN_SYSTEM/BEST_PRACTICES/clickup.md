## Em uma frase
ClickUp domina a **gestão visual de trabalho**: status customizáveis, filtros
poderosos e múltiplas visões (lista, board, tabela) do mesmo dado, para cada gestor ver o que precisa.

## Princípios a aprender
- **Status como sistema visual.** Estados claros, coloridos, customizáveis — leitura instantânea.
- **Filtros e agrupamentos poderosos.** Fatiar dados por qualquer dimensão sob demanda.
- **Múltiplas visões do mesmo dado.** Lista, board, tabela, calendário — o dado é um, a lente muda.
- **Bulk actions.** Operar em muitos itens de uma vez; eficiência de gestão.
- **Visão hierárquica** (espaço → pasta → lista → tarefa) para organizar escala.

## Como aplicar na Pralís
- ☀️ **Admin/relatórios (modelo principal):** status visual de colaboradores/trilhas
  (ativo/pendente/concluído/atrasado) + **filtros e agrupamentos** ricos.
- ☀️ **Dashboard:** múltiplas visões do mesmo dado — lista densa **ou** board de status
  **ou** tabela — para diferentes perguntas do gestor.
- ☀️ **Admin:** bulk actions em colaboradores (atribuir trilha, notificar) com eficiência.
- ☀️ **Admin:** visão hierárquica Gerente → Equipe → Colaborador (já no drill-down 360).

## NÃO copiar
- **Sobrecarga de features** / UI poluída — ClickUp peca por excesso; Pralís preza foco.
- Paleta **multicolor saturada** e densidade caótica — anti-calma premium da Pralís.
- "Tudo configurável" que paralisa; oferecer **defaults quentes e claros** primeiro.
